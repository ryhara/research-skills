# 論文読解 Skill

## 目的
与えられた論文 (PDF パス / arXiv URL / HTML) を読み込み、研究者が必要とする観点で構造化要約を作成する。

## 手順
1. 入力の種別を判別する
   - ローカル PDF パス → Read ツール (長い場合 `pages` 指定で分割)
   - arXiv URL → WebFetch で abstract ページを取得、必要に応じて PDF も取得
   - HTML URL → WebFetch
2. 以下の観点で抽出する:
   - **書誌情報**: タイトル / 著者 / 会議・ジャーナル / 年
   - **背景と課題**: 何を解こうとしているか、なぜ重要か
   - **提案手法**: コアアイデアを 3〜5 行で。必要なら数式・アルゴリズムを簡潔に
   - **実験設定**: データセット / ベースライン / 評価指標
   - **実験結果**: 数値を含めて定量的に
   - **限界・今後の課題**: 著者が明示したもの + 読者視点での批判
   - **関連研究との位置づけ**: 先行研究との差分・次に読むべき論文を示唆
3. 最後に「この論文を自分の研究で使うなら」という観点で 2〜3 個の示唆を添える

## 出力フォーマット
Markdown で以下のテンプレートに沿って出力する。

```markdown
# {タイトル}
**著者**: ... / **会議**: ... / **年**: ...

## 1. 背景と課題
## 2. 提案手法
## 3. 実験設定
## 4. 実験結果
## 5. 限界・今後の課題
## 6. 関連研究との位置づけ
## 7. 自分の研究への示唆
```

## 注意
- 本文に書かれていないことは推測せず「記載なし」と明記する
- 数式は LaTeX 表記で保存 (`$...$`)
- 長大な論文は章ごとに分割して読む
